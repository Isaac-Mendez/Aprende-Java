# Anatomía de un método (Parte 4)

En este apartado veremos cómo se elige el **nombre de un método**, algo que forma parte de la “gramática” de la programación. No es complicado, pero sí importante para mantener el código claro y fácil de entender.

---

## Cómo elegir un buen nombre

Antes de escribir el nombre de un método, debemos tener claro **qué hace**.  
El nombre debe describir su función de manera simple y directa.

Por ejemplo, si el método suma dos números, un buen nombre sería:

`sumarNumeros()`

La idea es evitar nombres confusos o que dificulten entender la funcionalidad del método.

---

## Regla importante para nombrar métodos

Un método **no puede comenzar por un número**.  
Los nombres válidos deben comenzar **siempre por una letra**.

### ✔ Ejemplos correctos:

`sumar() calcularEdad() mostrarMensaje()`

### ❌ Ejemplos incorrectos:

`1suma()      // No puede empezar por número 3calcular()`
