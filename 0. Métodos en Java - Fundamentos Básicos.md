## Introducción

Para avanzar en esta parte del curso, es importante tener unas bases mínimas: conocer la sintaxis de Java, entender los tipos de datos y tener algo de lógica. A partir de aquí empezaremos a crear pequeños programas y a estructurar nuestro código de forma profesional.

---

## ¿Qué es un método?

Un **método** es un bloque de código que realiza una tarea específica y **solo se ejecuta cuando es llamado**.  
Los métodos pueden llamarse desde la misma clase o desde otra, siempre que su modificador de acceso lo permita (por ejemplo, _public_, private: Estos son los permisos para que me entiendas mejor ).

En Java se llaman **métodos** y no _funciones_ porque **todo está dentro de una clase**.  
Una función puede existir fuera de clases en otros lenguajes, pero en Java no.

---

## ¿Para qué sirve aprender métodos?

Los métodos nos permiten:

- organizar mejor el código
    
- reutilizar instrucciones sin repetirlas
    
- dividir un problema grande en partes pequeñas
    
- mantener un proyecto ordenado y fácil de mantener
    

Imagina entrar a un proyecto grande: actualizar o crear nuevas funcionalidades normalmente significa crear **nuevos métodos** que agregan o modifican comportamientos del programa.

---

## ¿Qué tipos de métodos existen?

Podemos clasificarlos de varias formas, pero los principales son:

### ✔ Métodos que **no retornan nada**

Son los métodos cuyo tipo es **void**.  
Se usan cuando solo queremos ejecutar instrucciones sin obtener un valor.

Ejemplo típico: imprimir números, mostrar mensajes, modificar datos internos, etc.

```void contar() {     for (int i = 1; i <= 10; i++) {         System.out.println(i);     } }```

### ✔ Métodos que **retornan un valor**

Son métodos que devuelven un dato cuando finalizan.  
Pueden devolver ` int, String, boolean, etc`.

`int sumar(int a, int b) {     return a + b; }`

> **Importante:**  
> El `return` NO siempre termina el programa. Solo termina el método en el que está.
