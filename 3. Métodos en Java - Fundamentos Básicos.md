## Anatomía de un método (Parte 3)

Ya con lo que veremos ahora estamos casi finalizando. En este apartado explicaremos la palabra reservada `void` y también qué significa cuando un método **no** usa esta palabra.

## Palabra reservada `void`

Este concepto es muy sencillo de explicar: **cuando un método lleva `void`, significa que ese método no retorna ningún valor**. Es decir, ejecuta una acción, pero no devuelve nada al terminar.

Ejemplos de métodos que usarían `void`:

- Mostrar texto en pantalla
    
- Guardar datos
    
- Enviar un mensaje
    
- Imprimir una lista
    

## Métodos sin `void`

El otro caso que podemos encontrar es cuando el método **no** lleva la palabra reservada `void`. Esto significa que dicho método **sí debe retornar un valor**. Para devolver ese valor se utiliza la palabra clave `return`.

La diferencia importante es que **no se sustituye `void` por `return`**, sino que en lugar de `void` se coloca el **tipo de dato** que el método va a retornar. (Más adelante veremos esto con profundidad.)
